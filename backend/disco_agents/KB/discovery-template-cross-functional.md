# Discovery Template: Cross-Functional Initiatives

**Version:** 1.0
**Last Updated:** 2026-01-23

Use this template for initiatives where the core pattern is: "Multiple departments need to work together" or "Handoffs between teams" or "Requires cross-org alignment."

**Note:** This type has the highest failure rate. The biggest risk is assuming alignment exists when it doesn't.

## Required Questions (MUST answer before synthesis)

### RACI Clarity (CRITICAL)
- [ ] "Who is Responsible for [each major step]?"
- [ ] "Who is Accountable - the single decision-maker?"
- [ ] "Who needs to be Consulted before decisions?"
- [ ] "Who needs to be Informed after decisions?"

### Incentive Alignment (CRITICAL)
- [ ] "How is each team measured? What are their KPIs?"
- [ ] "Do those incentives align with this shared goal?"
- [ ] "What does each team gain if this succeeds? What do they lose?"

### Escalation & Governance
- [ ] "When teams disagree, who breaks ties?"
- [ ] "How are changes to the process approved?"
- [ ] "Who owns the end-to-end process?"

### Shared Metrics
- [ ] "What does success look like for the whole initiative?"
- [ ] "Can you all agree on how to measure that?"
- [ ] "How is credit or blame distributed?"

## Type-Specific Failure Patterns to Watch

| Pattern | Why Cross-Functional Initiatives Fail Here | Detection Questions |
|---------|-------------------------------------------|---------------------|
| **Assumed alignment** | "Everyone wants this to work" (they don't) | "What does each team lose if this succeeds?" |
| **No single owner** | "We're all responsible" = no one is | "Who is the single accountable person?" |
| **Incentive conflict** | Teams optimizing locally, not globally | "How are you measured? Does that align with this goal?" |
| **Escalation vacuum** | Disagreements fester with no resolution path | "When you disagree, who decides?" |
| **Committee death spiral** | Everyone needs to be consulted on everything | "Who can make decisions without a meeting?" |
| **Handoff blindness** | "I threw it over the wall, not my problem now" | "What happens at each handoff? Who owns the gap?" |

## Quantification Gate Checklist

Before synthesis, we MUST have:

- [ ] RACI: Complete matrix with sign-off from each team
- [ ] Incentives: How each team is measured (conflicts identified)
- [ ] Governance: Clear escalation path and decision rights
- [ ] Current state: Time/cost of handoffs and rework
- [ ] Shared metrics: Agreed definition of success
- [ ] Sponsor authority: Confirmation sponsor can enforce across teams

## Warning Signs (Red Flags)

Watch for these signals that suggest this initiative will struggle:

1. **"We're all aligned"** - Often means no one's tested it yet
2. **"It's a partnership"** - Often means no one is accountable
3. **"We just need better communication"** - Often means incentives conflict
4. **Meetings without decisions** - Governance gap
5. **Different definitions of success** - No shared metrics
6. **"That's not my job"** - Handoff ownership unclear
