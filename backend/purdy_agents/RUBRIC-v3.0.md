# PuRDy v3.0 Scoring Rubric

**Version:** 3.0
**Last Updated:** 2026-01-24
**Purpose:** Measure real outcomes, not document impressiveness

---

## The Core Shift

**Old Metrics (What We Stopped Measuring):**
- % quality score (105%, 112%) - Self-referential
- Word count / section presence - More isn't better
- Number of frameworks used (A3, 3M, VSM) - Sophistication ≠ clarity
- Checklist completion rate - Process compliance ≠ outcome achievement

**New Metrics (What We Started Measuring):**
- Decision velocity, action clarity, stakeholder conviction
- Surprise rate, root cause accuracy, risk prediction
- Time to confidence, output utilization, process improvements identified

---

## Tier 1: Did It Enable Action? (50% weight)

| Metric | How to Measure | Target | Score |
|--------|----------------|--------|-------|
| **Decision Velocity** | Days from synthesis → first decision | <7 days = 5, 8-14 = 3, 15+ = 1 | /5 |
| **Action Clarity** | Can stakeholder state next action in <30 sec? | Yes = 5, Partial = 3, No = 1 | /5 |
| **Stakeholder Conviction** | "I know what to do" rating (1-5 scale) | ≥4 = 5, 3 = 3, <3 = 1 | /5 |
| **Recommendation Adoption** | % of recommendations that became projects | ≥60% = 5, 40-59% = 3, <40% = 1 | /5 |
| **Blocker Resolution** | % of identified blockers that got addressed | ≥50% = 5, 30-49% = 3, <30% = 1 | /5 |

## Score Interpretation

| Score | Rating | Interpretation |
|-------|--------|----------------|
| 90-100 | Excellent | Outputs consistently enable decisions and improve processes |

## Quick Self-Assessment (Per Output)

Use this checklist immediately after generating an output:

### The 30-Second Test
- [ ] Can stakeholder state the leverage point after 30 seconds of reading?
- [ ] Is the first action clear enough to start Monday morning?
- [ ] Is the decision owner named?

### The Utilization Test
- [ ] What 40% of this would stakeholders skip? (If >40%, cut it)
- [ ] What sections would never be referenced in a meeting? (Cut them)

### The Systems Test
- [ ] Is a feedback loop visible (if system dynamics detected)?
- [ ] Does it show WHERE to intervene?

## Continuous Improvement Cadence

### Daily (5 min)
After each session, note:
- Did stakeholder understand what to do?
- What section did they skip to?
- What question wasn't answered?

### Weekly (15 min)
Review the week's signals:
- Patterns in what's working/not working
- Recurring gaps or confusions
- Sections consistently skipped

### Monthly (2 hours)
1. Review outcomes: Which initiatives moved forward? Which stalled?
2. Analyze patterns: What's the root cause of failures?
3. Prioritize: Pick 1-2 agent instruction changes
4. Implement: Update agents, test on next initiative

### Quarterly (half day)
1. Outcome audit: Recommendation adoption rate, root cause accuracy
2. Rubric validation: Are metrics actually predicting outcomes?
3. Major revisions: Significant agent instruction changes
4. Retrospective: What did we learn about improvement itself?

## What Good Looks Like (Examples)

### High Score Output Characteristics:
1. **Leverage point in first paragraph** - Stakeholder knows THE thing immediately
2. **Feedback loop visualized** - Clear why problem persists
3. **Actions numbered with owners** - No ambiguity about what's next
4. **<600 words before appendix** - Respects attention, forces prioritization
5. **Stakeholder rating ≥4** - "I know what to do"

### Low Score Output Red Flags:
1. **"Comprehensive" as a virtue** - Length ≠ value
2. **Multiple "equally important" recommendations** - No prioritization
3. **Persona briefs inline** - Bloat that's rarely used
4. **Framework names in output** (A3, VSM, 3M) - Process showing through
5. **No named decision owner** - Accountability vacuum
