# Conversations and Chat

Master the chat interface to get the most out of your interactions with Thesis.

## Starting Conversations

### New Conversation

1. Click **Chat** in the navigation bar
2. Click **New Conversation** or the **+** button in the sidebar
3. Type your first message and press Enter

### Starting from a Project

You can start a conversation directly from a project:

1. Go to **Projects** in the navigation bar
2. Find the project you want to work on
3. Click the **+** (plus) icon in the project's action row
4. This opens a new conversation already assigned to that project

This is the fastest way to start project-specific work.

### Conversation Context

Each conversation maintains its own context. Thesis remembers:

- Everything discussed in this conversation
- Your previous questions and requests
- Deliverables and content created
- Images generated

This means you can build on previous responses without repeating context.

**Example flow:**
1. "Help me design a training program for new managers"
2. "Add a section on giving feedback" (Thesis knows you're still talking about the manager program)
3. "Create an assessment for the feedback section" (Thesis understands the context)

## The Chat Interface

### Message Input

At the bottom of the screen:

- **Text field** - Type your message
- **Paperclip icon** - Attach documents
- **Send button** - Submit your message (or press Enter)

### Conversation Sidebar

On the left side:

- **Recent conversations** - Your latest chats
- **Search** - Find past conversations
- **New conversation** - Start fresh
- **Project filter** - Show only conversations from a specific project

### Message Display

Each message shows:

- **Your messages** - Aligned right, highlighted
- **Thesis's responses** - Aligned left
- **Images** - Displayed inline when generated
- **Timestamps** - When each message was sent

## Searching Conversations

### Finding Past Conversations

1. In the sidebar, click the **search** icon or field
2. Type keywords, topics, or phrases
3. Matching conversations appear in the list
4. Click to open any conversation

### Search Tips

- **Use specific terms** - "sales training assessment" finds more relevant results than "training"
- **Search deliverable types** - "facilitator guide" or "curriculum outline"
- **Client or project names** - If mentioned in conversations
- **Dates** - Recent conversations appear at the top by default

## Exporting Conversations

### Single Conversation Export

Save a conversation for documentation or sharing:

1. Open the conversation
2. Click the **export** icon (or menu → Export)
3. Choose your format
4. Download the file

### Export Formats

| Format | Best For |
|--------|----------|
| **Markdown** | Technical documentation, version control |
| **Text** | Simple sharing, email attachment |
| **PDF** | Formal documentation, printing |

### Bulk Export

Export multiple conversations at once:

1. Go to **Conversations** (if available in sidebar)
2. Select multiple conversations using checkboxes
3. Click **Export Selected**
4. Choose date range and format
5. Download the archive

## Conversation Management

### Naming Conversations

Conversations are automatically named based on the first message. To rename:

1. In the sidebar, hover over the conversation
2. Click the **edit** icon
3. Enter a new name
4. Press Enter or click Save

**Naming tips:**
- Use the client or project name
- Include the main topic
- Add dates for recurring work

### Deleting Conversations

To remove a conversation:

1. In the sidebar, hover over the conversation
2. Click the **delete** icon (trash can)
3. Confirm deletion

**Warning:** Deleted conversations cannot be recovered.

### Archiving Conversations

For conversations you want to keep but hide from the main view:

1. Open the conversation
2. Click the menu or options icon
3. Select **Archive**

Archived conversations are accessible through filters but don't clutter your active list.

## Effective Chat Strategies

### Be Specific

**Good:**
"Create a 2-hour workshop outline for customer service representatives on handling escalated complaints. Include 3 role-play activities and a knowledge check at the end."

**Less effective:**
"I need training on customer service."

### Provide Context

Share relevant details upfront:

- **Audience** - Who will use this content?
- **Constraints** - Time limits, format requirements, client preferences
- **Goals** - What should learners be able to do after training?
- **Industry** - Healthcare, retail, tech, etc.

### Build Iteratively

Thesis works well with back-and-forth refinement:

1. Start with a broad request
2. Review the initial response
3. Ask for specific modifications
4. Repeat until satisfied

**Example:**
1. "Create an outline for leadership training"
2. "Add more emphasis on emotional intelligence"
3. "Expand the section on difficult conversations"
4. "Add assessment questions for each module"

### Use Follow-up Questions

Don't start a new conversation for related requests. Continue the current one:

- "Can you add more detail to section 3?"
- "What about for remote teams?"
- "Create a one-page summary of this"

### Reference Your Documents

If you've uploaded relevant materials:

- "Using our competency framework, create an assessment..."
- "Based on our L&D strategy document, recommend..."
- "Apply our company's feedback model to this scenario"

## Handling Long Responses

For complex deliverables, Thesis may generate lengthy responses.

### Scrolling and Navigation

- Scroll within the message to read the full content
- Use your browser's Find function (Ctrl/Cmd+F) to locate specific content

### Copying Content

1. Select the text you want to copy
2. Right-click → Copy, or use Ctrl/Cmd+C
3. Paste into your document

For code blocks or formatted content, click the **copy** icon that appears on hover.

### Requesting Sections

If responses are too long, ask for sections:

- "Just give me the learning objectives for now"
- "Let's start with the first module only"
- "Summarize this in bullet points"

## Troubleshooting Chat Issues

### Response Taking Too Long

If Thesis seems stuck:

1. Wait up to 60 seconds for complex requests
2. If no response, try refreshing the page
3. Your message history is saved - you won't lose context

### Message Not Sending

If your message won't send:

1. Check your internet connection
2. Refresh the page
3. Try typing a shorter test message
4. Clear browser cache if issues persist

### Unexpected Responses

If Thesis's response seems off-topic:

- Rephrase your question with more context
- Start a new conversation if the context has drifted
- Be more explicit about what you need
