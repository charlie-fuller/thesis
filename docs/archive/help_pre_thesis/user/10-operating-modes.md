# Operating Modes

Thesis adapts to four distinct operating modes, each optimized for different types of L&D work.

## The Four Modes

| Mode | Purpose | Best For |
|------|---------|----------|
| **Coach** | Collaborative exploration | Brainstorming, refining ideas, problem-solving |
| **Developer** | Creating deliverables | Curricula, assessments, guides, materials |
| **Analyst** | Evaluating and measuring | Program evaluation, gap analysis, data review |
| **Advisor** | Strategic guidance | Stakeholder management, ROI planning, consulting |

## Coach Mode

### When to Use Coach Mode

Use Coach mode when you need to:
- Explore ideas and possibilities
- Think through challenges
- Get a thought partner's perspective
- Refine concepts before development
- Work through ambiguity

### What Coach Mode Does

In Coach mode, Thesis:
- Asks clarifying questions
- Offers multiple perspectives
- Challenges assumptions constructively
- Helps you discover your own solutions
- Supports iterative thinking

### Coach Mode Examples

**Brainstorming:**
```
User: "I need to improve our onboarding program but I'm not sure where to start"

Thesis (Coach): "Let's explore this together. What signals are telling you the
current program needs improvement? Are you seeing specific issues with new hire
performance, feedback from managers, or attrition patterns?"
```

**Refining Ideas:**
```
User: "I want to add gamification to our compliance training"

Thesis (Coach): "That's an interesting direction. Before we dive in—what outcome
are you hoping gamification will achieve? Higher completion rates? Better
retention? More engagement? The specific goal will shape the approach."
```

**Problem-Solving:**
```
User: "Managers aren't sending their teams to our training sessions"

Thesis (Coach): "That's a common challenge. Let's think about what might be
driving that behavior. Is it scheduling conflicts, perceived value, manager
buy-in, or something else? What have you observed about the managers who
do support training?"
```

### Tips for Coach Mode

- Be open to questions—Coach mode is conversational
- Share your constraints and context
- Don't expect immediate deliverables
- Use it early in projects before development

## Developer Mode

### When to Use Developer Mode

Use Developer mode when you need to:
- Create polished, client-ready materials
- Generate specific deliverables
- Produce assessments, guides, or curricula
- Build content efficiently

### What Developer Mode Does

In Developer mode, Thesis:
- Produces complete, formatted outputs
- Follows instructional design best practices
- Applies the DDLD framework
- Creates content ready for delivery or minor editing
- Maintains consistent quality and structure

### Developer Mode Examples

**Curriculum Development:**
```
User: "Create a 3-module curriculum on consultative selling for B2B sales reps"

Thesis (Developer): [Generates complete curriculum including:]
- Module structure and timing
- Learning objectives (SMART format)
- Content outline for each module
- Activities and practice exercises
- Assessment checkpoints
- Facilitator notes
```

**Assessment Creation:**
```
User: "Build a competency assessment for project managers on stakeholder
communication"

Thesis (Developer): [Generates assessment including:]
- Scenario-based questions
- Skill evaluation criteria
- Rating scales with behavioral anchors
- Instructions for assessors
- Scoring guide
```

**Facilitator Guide:**
```
User: "Write a facilitator guide for the feedback workshop activity"

Thesis (Developer): [Produces guide including:]
- Activity purpose and objectives
- Time and materials needed
- Step-by-step facilitation instructions
- Discussion questions
- Common challenges and how to handle them
- Variations for different group sizes
```

### Tips for Developer Mode

- Be specific about format and length expectations
- Provide audience details
- Mention any templates or standards to follow
- Include constraints (time, resources, technology)

## Analyst Mode

### When to Use Analyst Mode

Use Analyst mode when you need to:
- Evaluate existing programs
- Identify performance gaps
- Review training effectiveness
- Analyze data or feedback
- Conduct needs assessments

### What Analyst Mode Does

In Analyst mode, Thesis:
- Applies systematic evaluation frameworks
- Identifies gaps and opportunities
- Provides evidence-based recommendations
- Connects findings to learning science
- Structures analysis for decision-making

### Analyst Mode Examples

**Program Evaluation:**
```
User: "Evaluate our leadership development program based on these participant
feedback scores and manager assessments"

Thesis (Analyst): [Provides analysis including:]
- Summary of key findings
- Strengths identified in the data
- Areas needing improvement
- Comparison to industry benchmarks
- Prioritized recommendations
- Suggested measurement improvements
```

**Gap Analysis:**
```
User: "We have these competencies defined but employees aren't demonstrating them.
What's the gap?"

Thesis (Analyst): [Delivers analysis including:]
- Framework for understanding the gap (knowledge, skill, motivation, environment)
- Questions to diagnose root cause
- Data to gather for confirmation
- Intervention options based on gap type
- Measurement approach
```

**Needs Assessment:**
```
User: "Help me analyze these interview transcripts from stakeholders about
training needs"

Thesis (Analyst): [Produces analysis including:]
- Theme identification across interviews
- Consensus and divergent views
- Priority needs ranked by impact
- Evidence supporting each finding
- Recommended next steps
```

### Tips for Analyst Mode

- Provide data or evidence when available
- Share context about what you're evaluating
- Be clear about what decisions the analysis will inform
- Ask for specific frameworks if you have preferences

## Advisor Mode

### When to Use Advisor Mode

Use Advisor mode when you need to:
- Navigate stakeholder dynamics
- Plan ROI strategies
- Think strategically about L&D
- Get consulting-style guidance
- Prepare for executive conversations

### What Advisor Mode Does

In Advisor mode, Thesis:
- Provides strategic recommendations
- Anticipates stakeholder questions
- Frames L&D in business terms
- Offers change management guidance
- Helps with influence and buy-in

### Advisor Mode Examples

**Stakeholder Management:**
```
User: "I need to get executive buy-in for a major LMS change"

Thesis (Advisor): [Provides guidance including:]
- Key stakeholder concerns to anticipate
- Business case framing
- Risk mitigation messaging
- Recommended presentation approach
- Objection handling strategies
- Timeline and milestone communication
```

**ROI Planning:**
```
User: "How do I justify the cost of this management training program?"

Thesis (Advisor): [Delivers strategy including:]
- ROI calculation framework
- Metrics to track
- Data collection approach
- Baseline measurement needs
- Expected timeline for results
- Executive communication strategy
```

**Strategic Consulting:**
```
User: "Our company is going through a merger. What should L&D be doing?"

Thesis (Advisor): [Offers perspective including:]
- Priority areas for L&D during mergers
- Quick wins vs. long-term initiatives
- Cultural integration through learning
- Communication strategy
- Risk areas to address
- Stakeholder alignment approach
```

### Tips for Advisor Mode

- Share organizational context and politics
- Be clear about who the decision-makers are
- Describe the constraints you're working within
- Ask for specific deliverables (talking points, presentations, plans)

## Switching Between Modes

### Mode Selection

You can explicitly request a mode:

```
"In Coach mode, help me think through this training design challenge"
"Switch to Developer mode and create a facilitator guide"
"As an Analyst, evaluate this training data"
"Give me Advisor-level guidance on this stakeholder situation"
```

### Natural Mode Shifts

Thesis often shifts modes based on your request:

- Questions about "what should I do?" → Coach
- Requests to "create" or "build" → Developer
- Requests to "evaluate" or "analyze" → Analyst
- Questions about "how do I convince" or "strategy for" → Advisor

### Combining Modes

Many projects naturally flow through modes:

1. **Coach** → Explore the challenge and refine the approach
2. **Analyst** → Assess current state and identify gaps
3. **Developer** → Create the training solution
4. **Advisor** → Plan the rollout and stakeholder communication

## Mode Selection Guide

| If You Need To... | Use This Mode |
|-------------------|---------------|
| Think through a problem | Coach |
| Explore options | Coach |
| Get questions to consider | Coach |
| Create training materials | Developer |
| Build assessments | Developer |
| Write facilitator guides | Developer |
| Evaluate program effectiveness | Analyst |
| Identify skill gaps | Analyst |
| Analyze feedback data | Analyst |
| Plan stakeholder communication | Advisor |
| Build business cases | Advisor |
| Navigate organizational politics | Advisor |
