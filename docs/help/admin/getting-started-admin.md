# Getting Started as an Administrator

Welcome to the Thesis admin guide! This document will help you understand your role as an administrator and how to navigate the admin interface effectively.

## What Can Administrators Do?

As a Thesis administrator, you have access to powerful tools for managing the platform:

- **Manage users** - Create accounts, assign roles, and monitor user activity
- **Oversee documents** - View, download, and manage all uploaded documents across clients
- **Monitor conversations** - Review chat histories for quality assurance and insights
- **Customize branding** - Configure colors, logos, typography, and overall theme
- **Configure AI behavior** - Map documents to system instruction templates
- **Track performance** - View KPIs and analytics on platform usage

## The Admin Dashboard

When you log in as an administrator, you'll see the **Admin Dashboard** with three main tabs:

### Overview Tab

The **Overview** tab provides at-a-glance information:

- **Quick Stats** - Displays four key metrics:
  - **Total Users** - Number of registered users
  - **Conversations** - Total conversation count
  - **Documents** - Number of uploaded documents
  - **Messages** - Total messages exchanged

- **Recent Activity** - Shows the latest actions across the platform

### KPI Dashboard Tab

The **KPI Dashboard** tab displays performance metrics based on the Bradbury Impact Loop methodology:

- **Show metrics for:** dropdown - Filter by **All Users** or select a specific user
- **Ideation Velocity** - Measures idea generation and creative output
- **Correction Loop** - Tracks iterative refinement patterns
- **Correction Loop Efficiency Over Time** - Visualizes improvement trends
- **Keyword Trends** - Shows topic patterns in conversations

### Usage Analytics Tab

The **Usage Analytics** tab provides detailed usage patterns and trends over time.

## Admin Navigation

The admin interface has six main sections accessible from the top navigation bar:

1. **Dashboard** - Your home base with stats, KPIs, and activity
2. **Users** - Manage user accounts and permissions
3. **Documents** - View and manage all uploaded documents
4. **Conversations** - Review and export conversation histories
5. **Theme** - Customize the platform's appearance and branding
6. **Help System** - Manage help documentation and monitor help effectiveness

## View Mode Toggle

Administrators can experience the platform as a regular user would by switching to "User View Mode."

### Switching to User View

1. Click on your user menu (avatar/name) in the top navigation bar
2. Find the **View Mode** toggle or option
3. Select **User View** to switch perspectives
4. The navigation will change to show user pages: Chat, Projects, My Impact

### What Changes in User View

When you switch to User View:
- **Navigation changes** - You see Chat, Projects, and My Impact instead of admin pages
- **Interface simplifies** - You see exactly what regular users see
- **Your data is preserved** - You're still logged in as yourself
- **Admin access remains** - Toggle back anytime to return to admin mode

### What You Can Do in User View

- Start conversations with Thesis as a user would
- Upload and manage documents from the user perspective
- Access the user help chat for user-focused assistance
- Test projects and impact dashboard features

### What You Cannot Do in User View

- Access admin pages (Dashboard, Users, Documents, Conversations, Theme, Help System)
- View other users' data
- Modify system settings

### Returning to Admin View

1. Click on your user menu again
2. Toggle the view mode back to **Admin View**
3. Full admin navigation and capabilities are restored

### When to Use View Mode

This feature is valuable for:
- **Testing user experience** - See exactly what users see
- **Troubleshooting issues** - Reproduce problems users report
- **Training and demos** - Show the user interface without a separate account
- **Quality assurance** - Verify user-facing features work correctly

## Using the Admin Help Chat

Administrators have access to a specialized help chat that provides guidance on admin tasks.

### Accessing Admin Help

1. Look for the help icon or panel toggle in the top navigation bar
2. Click to open the **Help Chat** sidebar on the right side of the screen
3. The help chat appears alongside your current admin page

### What Admin Help Can Do

The admin help chat is powered by the same RAG system you manage in the Help System page. It can:
- Answer questions about admin features and workflows
- Explain how to use specific admin tools
- Provide step-by-step guidance for common tasks
- Reference the admin documentation you've indexed

### Admin vs User Help

The help chat automatically knows you're an admin and:
- Searches admin-specific documentation
- Provides answers relevant to administrative tasks
- Can reference both admin and user documentation
- Tracks conversations separately in help analytics (shown as "Admin" context)

### Tips for Using Admin Help

- Ask specific questions for better answers (e.g., "How do I reindex help documents?" vs "help with documents")
- The help chat remembers context within a conversation
- Start a new conversation for unrelated topics
- If answers seem incomplete, check if relevant documentation exists in the Help System

## Your First Admin Tasks

Here's a recommended order for getting started:

### 1. Explore the Dashboard

Navigate to **Dashboard** and review the **Overview** tab to understand current platform usage.

### 2. Check User List

Go to **Dashboard** → **Users** to see all registered users and their roles.

### 3. Review Theme Settings

Visit **Dashboard** → **Theme** to understand current branding and customization options.

### 4. Browse Documents

Navigate to **Dashboard** → **Documents** to see what content has been uploaded.

### 5. Sample Some Conversations

Check **Dashboard** → **Conversations** to review how users are interacting with Thesis.

## Key Administrator Responsibilities

### User Management
- Create new user accounts when onboarding team members
- Assign appropriate roles (**Admin** or **User**)
- Resend invitation emails when needed
- Export user chat histories for analysis

### Content Oversight
- Monitor document processing status
- Remove outdated or incorrect documents
- Ensure documents are properly processed

### Quality Assurance
- Review conversations for training quality
- Identify common user questions or issues
- Export conversations for further analysis

### Platform Customization
- Configure branding to match organizational identity
- Set up color palettes and typography
- Upload and manage logos

### AI Configuration
- Map documents to instruction template slots
- Regenerate system instructions after mapping changes
- Ensure AI responses align with organizational guidelines

### Help System Management
- Monitor help documentation indexing status
- Add and reindex help documentation
- Review help analytics to identify documentation gaps
- Export help conversations for analysis
