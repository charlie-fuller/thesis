# Project Management

Organize your conversations and work by client, initiative, or program using Thesis's project system.

## Why Use Projects?

Projects help you:

- **Stay organized** - Group related conversations together
- **Track progress** - See where each initiative stands
- **Find work faster** - Filter conversations by project
- **Maintain context** - Keep client or program-specific work separate

Without projects, all your conversations exist in one long list. With projects, you can separate "Acme Corp Sales Training" from "Internal Leadership Development" from "Client XYZ Onboarding."

## Creating a Project

### From the Projects Page

1. Click **Projects** in the navigation bar
2. Click **New Project** or the **+** button
3. Fill in the project details:
   - **Title** - A clear, descriptive name
   - **Description** - Optional context about the project
   - **Phase** - Current stage of the work
   - **Status** - Active, Archived, or Complete
4. Click **Create Project**

### Quick Project Creation

When starting a new conversation, you can create a project on the fly:

1. Start a new conversation
2. Before sending your first message, click **Assign to Project**
3. Select **Create New Project**
4. Enter a project name
5. Continue with your conversation

## Project Details

### Project Title

Choose clear, specific names that distinguish projects:

**Good examples:**
- "Acme Corp - Q1 2024 Sales Enablement"
- "Internal Leadership Development Program"
- "Healthcare Compliance Training Refresh"

**Less helpful:**
- "Training Project"
- "Client Work"
- "New Program"

### Project Description

Use the description field for context you'll want later:

- Client or stakeholder information
- Key objectives or deliverables
- Timeline or deadline notes
- Links to external resources

### Project Phases

Track where each project stands in the L&D lifecycle:

| Phase | Description | Typical Activities |
|-------|-------------|-------------------|
| **Analysis** | Understanding the need | Needs assessments, stakeholder interviews, data gathering |
| **Design** | Planning the solution | Learning objectives, curriculum outlines, content planning |
| **Development** | Creating the content | Writing materials, creating assessments, generating visuals |
| **Implementation** | Deploying the solution | Facilitator prep, pilot sessions, rollout planning |
| **Evaluation** | Measuring the results | Assessments, feedback analysis, ROI measurement |

These phases align with the ADDIE model commonly used in L&D.

### Project Status

| Status | Meaning | Use When |
|--------|---------|----------|
| **Active** | Currently being worked on | Project is in progress |
| **Complete** | Finished and delivered | All work is done, for reference only |
| **Archived** | Set aside | Paused, cancelled, or for long-term storage |

## Managing Projects

### Viewing All Projects

1. Click **Projects** in the navigation bar
2. See all your projects with status indicators
3. Sort by name, date, or status

### Editing a Project

1. Go to **Projects**
2. Click on the project you want to edit
3. Click **Edit** or the edit icon
4. Update any fields
5. Click **Save Changes**

### Changing Project Phase

As your work progresses, update the phase:

1. Open the project
2. Click the phase indicator or **Edit**
3. Select the new phase
4. Save your changes

This helps you track progress across all your initiatives.

### Archiving a Project

When a project is complete or on hold:

1. Open the project
2. Change the status to **Archived** or **Complete**
3. Save changes

Archived projects remain accessible but are hidden from the default view. Use filters to show archived projects when needed.

### Deleting a Project

**Warning:** Deleting a project removes it permanently.

1. Open the project
2. Click **Delete** or the delete icon
3. Confirm deletion

**What happens:**
- The project is permanently removed
- Conversations in the project remain but become unassigned
- This cannot be undone

## Organizing Conversations

### Assigning Conversations to Projects

**When starting a new conversation:**
1. Click **New Conversation**
2. Select a project from the dropdown
3. Begin your conversation

**For existing conversations:**
1. Open the conversation
2. Click the project indicator or **Assign to Project**
3. Select the target project
4. Confirm the assignment

### Moving Conversations Between Projects

1. Open the conversation
2. Click the current project name
3. Select a different project
4. Confirm the change

### Viewing Project Conversations

1. Go to **Projects**
2. Find the project you want to view
3. Click on the conversation count row (e.g., "3 conversations")
4. The conversation list expands in place, showing all conversations in that project
5. Click any conversation title to open it
6. Click the row again to collapse the list

**Starting a new conversation in a project:**
- Click the **+** (plus) icon button in the project's action row
- This creates a new conversation and automatically assigns it to that project

### Filtering by Project

In the conversation sidebar:

1. Click the **filter** icon
2. Select a project
3. Only conversations from that project appear

## Project Statistics

Each project displays activity metrics:

| Metric | Description |
|--------|-------------|
| **Conversations** | Number of conversations in the project |
| **Last Updated** | Most recent activity timestamp |
| **Phase** | Current stage of work |
| **Status** | Active, Complete, or Archived |

Use these metrics to quickly assess project activity and identify stale projects.

## Best Practices

### Project Granularity

**Too broad:**
- "All Client Work 2024" (contains everything)

**Too narrow:**
- "Acme Module 1 Lesson 2" (creates too many projects)

**Just right:**
- "Acme Corp Sales Training Program" (one initiative, multiple conversations)

### Naming Conventions

Establish patterns for your organization:

- `[Client] - [Program Name]`
- `[Year] [Department] [Initiative]`
- `[Client Code] - [Project Type] - [Description]`

### Regular Maintenance

- **Weekly:** Update phases for active projects
- **Monthly:** Archive completed projects
- **Quarterly:** Review and clean up stale projects

### Using Projects with Document Scope

When you assign a conversation to a project, you can also scope document search to project-relevant materials. This helps Thesis focus on the most relevant context.
