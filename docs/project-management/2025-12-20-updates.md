## Analytics & KPI Improvements

### Output Activity KPI - Copy Event Tracking
Added frontend tracking for the Output Activity KPI to measure when users produce useable output:

**Files Changed:**
- `frontend/components/ChatMessage.tsx` - Added copy event tracking
- `frontend/components/ChatInterface.tsx` - Pass conversation/message IDs to ChatMessage

**How it works:**
- When users copy content (50+ characters) from assistant messages, the frontend calls `/api/kpis/copy-event/{conversationId}`
- Tracks both code block copies (via Copy button) and text selection copies (via Ctrl+C/Cmd+C)
- Backend marks the conversation as having "useable output" for the Output Activity KPI
- Only significant copies (50+ chars) are tracked to filter out noise

**Implementation Details:**
- Added `conversationId` and `messageId` props to `ChatMessage` component
- `CodeBlock` component now tracks copy button clicks
- `handleCopyEvent` callback tracks text selection copies via `onCopy` event
- Non-blocking: copy failures don't affect the copy-to-clipboard functionality

### Ideation Velocity Chart Display
- Changed Ideation Velocity card from monthly to weekly display
- File: `frontend/app/admin/page.tsx`
- Changed `timePeriod="month"` to `timePeriod="week"` for more granular trending

### Output Activity KPI - Replaced Correction Loop Efficiency
Replaced the "Correction Loop Efficiency" metric with "Output Activity" on the admin KPI dashboard. The correction loop metric was designed for short, task-oriented interactions, but Thesis's methodology-driven conversations naturally involve more turns for deeper engagement.

**New Metric: Output Activity**
- Measures conversations where users produced tangible deliverables
- Tracks when users copy content, confirm usefulness, or otherwise indicate valuable output
- Shows output rate as a percentage of total conversations
- Includes breakdown by detection method (copy events, keyword confirmation, etc.)
- Weekly trend data for monitoring platform effectiveness

**Files Changed:**
- `backend/api/routes/kpis.py` - Added `/api/kpis/output-activity` endpoint
- `frontend/components/OutputActivityCard.tsx` - New component (created)
- `frontend/app/admin/page.tsx` - Replaced CorrectionLoopCard with OutputActivityCard

**Status Thresholds:**
- Strong (≥50%): Platform effectively producing deliverables
- Moderate (25-49%): Users producing materials, room for improvement
- Building (<25%): Early adoption or exploration phase

## Help Documentation In-App Editing

### New Feature: Edit Help Documents from Admin Panel
Added the ability to edit help documentation directly within the application's admin panel, eliminating the need to manually edit files and re-deploy.

**Backend Changes (`backend/api/routes/admin.py`):**
- Added `GET /api/admin/help-documents/{document_id}` endpoint to fetch a single document with its full content for editing
- Added `PUT /api/admin/help-documents/{document_id}` endpoint to update document title and/or content
- PUT endpoint automatically re-indexes the document after saving:
  - Deletes existing chunks for the document
  - Re-chunks the updated content (1000 char chunks, 200 char overlap)
  - Generates new embeddings via Voyage AI
  - Inserts new chunks with proper role access and metadata
- Added missing `Request` import (fixed startup crash)

**Frontend Changes (`frontend/app/admin/help-system/page.tsx`):**
- Added "Edit" button to both User and Admin document tables
- Created edit modal with:
  - Title input field
  - Markdown content textarea (monospace font, 96 lines height)
  - Live word count display
  - Save & Reindex button with loading state
- Added loading overlay when opening documents for edit
- Client-side validation (title min 3 chars, content min 50 chars)
- Auto-refreshes document list and status after successful save

**Workflow:**
1. Navigate to Admin → Help System → Documents tab
2. Click "Edit" on any document
3. Modify title and/or markdown content
4. Click "Save & Reindex" - document is saved and vector embeddings are regenerated
5. Changes are immediately available to the RAG help system
