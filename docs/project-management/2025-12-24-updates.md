# Thesis Updates - December 24, 2025

## System Instructions Versioning Feature

### Overview
Implemented a complete version control system for Thesis's global system instructions. This allows admins to manage, compare, and roll back system instruction versions with full audit trails.

### Database Migration
Created new database schema to support versioning:

**New Table: `system_instruction_versions`**
| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID | Primary key |
| `version_number` | VARCHAR(20) | Semantic version (e.g., "1.3", "2.0-beta") |
| `content` | TEXT | Full system instructions content |
| `storage_path` | TEXT | Optional file path reference |
| `file_size` | BIGINT | Size in bytes |
| `status` | VARCHAR(20) | "active" or "archived" |
| `is_active` | BOOLEAN | Whether this is the current active version |
| `version_notes` | TEXT | Changelog/description of changes |
| `created_by` | UUID | User who created the version |
| `activated_by` | UUID | User who activated the version |
| `created_at` | TIMESTAMPTZ | Creation timestamp |
| `activated_at` | TIMESTAMPTZ | Activation timestamp |
| `metadata` | JSONB | Additional metadata (original filename, etc.) |

**New Column on `conversations`:**
- `system_instruction_version_id` - Links each conversation to the version used when it was created

**Constraints:**
- Unique partial index ensures only one `is_active = TRUE` version at a time
- Version number format validated via regex: `^[0-9]+\.[0-9]+(-.+)?$`

### Backend API Endpoints

**New Route File:** `backend/api/routes/system_instructions.py`

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/admin/system-instructions/versions` | GET | List all versions with metadata |
| `/api/admin/system-instructions/versions/active` | GET | Get the current active version |
| `/api/admin/system-instructions/versions/{id}` | GET | Get specific version with full content |
| `/api/admin/system-instructions/upload` | POST | Upload new version (file + version number) |
| `/api/admin/system-instructions/versions/{id}/activate` | POST | Activate a version for new chats |
| `/api/admin/system-instructions/versions/compare` | POST | Generate unified diff between two versions |
| `/api/admin/system-instructions/versions/compare/summary` | POST | Generate AI-powered narrative summary |
| `/api/admin/system-instructions/versions/{id}/changelog` | GET | Fetch changelog file if exists |
| `/api/admin/system-instructions/versions/{id}` | PATCH | Update version notes |
| `/api/admin/system-instructions/versions/{id}` | DELETE | Delete a version (cannot delete active) |
| `/api/admin/system-instructions/versions/{id}/archive` | POST | Archive a version (soft delete) |

**Key Features:**
- File upload accepts `.txt` and `.xml` files up to 50MB
- Version activation invalidates all system instruction caches
- Conversation count tracking per version
- User attribution for creation and activation

### Frontend Admin Page

**New Page:** `/admin/system-instructions`

**Tab Navigation:**
1. **Active Version** - View the current active system instructions with metadata
2. **Version History** - List all versions with actions (View, Activate, Delete)
3. **Upload New** - Upload a new version with version number and notes
4. **Compare Versions** - Side-by-side comparison with collapsible panels

**Compare Versions Feature:**
Three collapsible panels showing:
1. **AI Summary** - Claude-generated narrative analysis of changes
   - Overview of what changed
   - Key changes as bullet points
   - Impact assessment on Thesis's behavior
   - Recommendations for testing
2. **Line-by-Line Diff** - Traditional unified diff with color coding
   - Green highlighting for additions
   - Red highlighting for deletions
   - Blue highlighting for context markers
3. **Version Changelogs** - Displays changelog files if they exist
   - Looks for `v{X.Y}-changelog.md` or `v{X.Y}-update-log.md`
   - Shows changelogs for both compared versions

**UI Components:**

### AI Summary Generation

Uses Claude API to analyze diffs and generate human-readable summaries:

```python
model="claude-sonnet-4-20250514"
max_tokens=1500
```

**Prompt Structure:**
1. Version information (numbers and dates)
2. Version notes from both versions
3. Statistics (additions/deletions)
4. Unified diff (truncated to 500 lines for token limits)
5. Structured response format request

### Changelog File Detection

The system automatically detects and displays changelog files:

**File Pattern Matching:**
- `v{version}-changelog.md` (e.g., `v1.3-changelog.md`)
- `v{version}-update-log.md` (e.g., `v1.1-update-log.md`)
- `v{version with dashes}-changelog.md` (e.g., `v1-3-changelog.md`)

**Location:** `docs/system-instructions/`
