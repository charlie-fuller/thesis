## Key Decisions Made

### 1. Architecture Change: Added Insight Extractor

**Before (v3.0):**
```
Triage → Discovery Planner → Coverage Tracker → Synthesizer → Tech Evaluation
```

**After (v4.0):**
```
Triage → Discovery Planner ↔ Coverage Tracker (loop) → Insight Extractor → Synthesizer → Tech Evaluation
```

**Rationale:** Synthesizer was processing raw transcripts (64K+ chars) AND creating decision documents. Split the work: Insight Extractor distills transcripts into structured insights, Synthesizer focuses on decision enablement.

### 2. Discovery Loop

Discovery Planner and Coverage Tracker work iteratively:
- Run Coverage Tracker during workshop breaks
- Run Coverage Tracker after each session
- If gaps found → return to Discovery Planner for follow-up
- If READY → proceed to Insight Extractor

**Rationale:** Discovery doesn't happen once. Real-world workshops need mid-course correction.

### 3. Consulting Quality Bar

All agents held to: "Would a PwC partner put their name on this?"

Internal customers expect Big 4 quality (Accenture, PwC level). Each agent output should be client-ready without editing.

### 4. Word Limits (Guidance, Not Enforced)

| Agent | Target | Actual Observed |
|-------|--------|-----------------|
| Triage | 250 | ~300 |
| Discovery Planner | 350 | 850-1200 |
| Coverage Tracker | 280 | ~400 |
| Insight Extractor | 500 | ~1000 |
| Synthesizer | 500 | 900-1400 |
| Tech Evaluation | 550 | ~800 |

**Finding:** Agents ignore strict limits and self-regulate to appropriate length. Quality is solid.
