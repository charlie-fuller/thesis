Goal

Reduce cold start time, eliminate unnecessary work at startup, and optimize the three hot paths (project_agent, kraken, taskmaster) that handle 99% of user traffic. Previously, all 6 schedulers + vault watcher started synchronously, blocking health check readiness. All 22 agent classes were imported eagerly. Conversation history loaded unbounded.

2 — Phase 1: Lazy Agent Loading

Done backend/agents/__init__.py

  • Eager imports: BaseAgent, AgentContext, AgentResponse, AgentRouter, RoutingDecision, factory functions
  • 17 agent classes lazy-loaded via __getattr__ + importlib.import_module()
  • Cached in globals() after first access — no repeated import cost

Done backend/agents/agent_factory.py

  • All 22 agent class imports replaced with _get_agent_class() registry
  • create_coordinator() and create_specialist() use lazy registry
  • Agent modules only loaded when that specific agent is requested
3 — Phase 2: Deferred Schedulers

New backend/main.py — lifespan

  • 5 schedulers wrapped in _deferred_schedulers() with 30s asyncio.sleep
  • Vault watcher deferred separately to 60s with initial_sync=False
  • sync_scheduler (Google Drive) removed entirely — packages already removed
  • Health check responds immediately — no scheduler dependency

4 — Phase 3: Hot Path Optimizations

New backend/api/routes/chat.py — conversation history limit

  • 3 conversation history fetches now use .limit(50)
  • Uses desc=True + .reverse() to get the most recent 50 messages in chronological order
  • Affected endpoints: non-streaming chat, streaming chat, dig-deeper
  • Section-expand endpoint already had .limit(10) — unchanged

New backend/agents/taskmaster.py — task query limit

  • Added .limit(200) + .order("due_date") to project_tasks query
  • Ensures soonest-due tasks are always included within the limit
  • Prevents full table scan on growing task history

Confirmed Instruction cache working

  • chat_agent_service.py line 114: self._instruction_cache: dict[str, str] = {}
  • Checks cache before DB lookup (line 142-143)
  • Caches on first fetch (line 183-184)
  • All 3 hot agents benefit: project_agent, kraken, taskmaster
