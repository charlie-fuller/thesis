Stage Details
Discovery

Discovery Guide v1.4

Validates whether the problem is worth solving using a 4-criteria framework.

Inputs
  • Linked documents & folders
  • Investigation framing (optional)
Outputs
  • GO / NO-GO / INVESTIGATE decision
  • Discovery session plan
  • Suggested throughline framing
  • Stakeholder candidates
Intelligence

Insight Analyst v1.2

Deep analysis of all documents to extract evidence-backed insights and system patterns.

Inputs
  • All initiative documents
  • Discovery outputs
  • Throughline context
Outputs
  • Decision document with recommendation
  • Leverage point & system dynamics
  • Evidence table with quotes
  • Hypothesis evidence mapping
Synthesis

Initiative Builder v1.2

Clusters insights into actionable initiative bundles with scoring.

Inputs
  • Insight Analyst outputs
  • Evidence & patterns
Outputs
  • Proposed initiative bundles
  • IFU scores per bundle
  • Complexity tier (Light/Med/Heavy)
  • Dependencies & stakeholder map
Convergence

Requirements Generator v1.4

Produces comprehensive output documents from approved initiatives.

Inputs
  • Approved initiative bundles
  • All prior stage outputs
Outputs
  • PRD, Evaluation, or Decision doc
  • Throughline resolution table
  • State changes & next actions
  • Tool & platform recommendations

Discovery Agent: Three Operating Modes (Auto-Detected)
Triage Mode
When: No prior Discovery outputs exist

Validates whether the problem is worth solving.

  • 4-Criteria Gate: Is it real? Costly? Solvable? Ours?
  • Five Whys root cause analysis
  • GO / NO-GO / INVESTIGATE decision
  • Extracts suggested framing (problem statements, hypotheses, gaps, KPIs)
Planning Mode
When: Triage was GO, but no session transcripts

Designs human-led discovery sessions.

  • Max 5 sessions with agendas
  • Quantification questions
  • Gap-targeted interview questions
  • Participant and sponsor identification
Coverage Mode
When: Session transcripts exist

Assesses discovery completeness after each session.

  • Status: READY / GAPS / BLOCKED
  • Absence reports (what's missing)
  • "Why This Matters" context
  • Run after EVERY discovery session
Investigation Framing (Throughline)

The throughline threads through every DISCO stage, giving agents structured context. Each stage maps its outputs back to these items, and Convergence resolves them.

Problem Statements
ps-1, ps-2, ps-3 ...

Defines what problem the initiative is investigating. Agents reference these in their analysis.

Hypotheses
h-1, h-2, h-3 ...

Testable assumptions, beliefs, or predictions. Typed as: assumption, belief, or prediction.

Gaps
g-1, g-2, g-3 ...

Known unknowns to investigate. Categorized as: data, people, process, or capability.

Throughline Resolution (Convergence): The final stage produces a resolution table showing each hypothesis as confirmed / refuted / inconclusive and each gap as addressed / unaddressed / partially addressed, with evidence citations. Users can override agent-assigned statuses with annotations.
Three Ways to Define a Throughline
Agent-Extracted
Recommended

Run Discovery Guide on linked documents. Agent analyzes content and suggests framing items in a review panel.

  • Accept all, selectively, or dismiss
  • Automatic during Triage mode
Chat-Based
 

Chat with the Discovery Agent. Ask "Help me frame this discovery." Agent proposes items as interactive cards.

  • Select items to keep
  • "Apply to Discovery" button
Manual
 

Expand "Investigation Framing" in the create/edit modal. Add items directly using + Add buttons.

  • Auto-assigned IDs (ps-1, h-1, g-1)
  • Full control over framing
Checkpoint Gates (Human Review Required)
1 Ready to Execute Discovery?

After Discovery Guide, before Insight Analyst

  • Triage decision is GO
  • Sessions planned with agendas
  • Participants identified
  • Review suggested framing
2 Ready for Initiative Bundles?

After Insight Analyst, before Initiative Builder

  • Decision document quality OK
  • Leverage point makes sense
  • Evidence supports conclusions
  • No critical gaps
3 Ready for Document Generation?

After Initiative Builder, before Requirements Generator

  • Bundle definitions clear
  • IFU scoring accurate
  • Dependencies mapped
  • Bundles approved/rejected/merged
4 Ready for Operationalization?

After Requirements Generator, before export/creation

  • Document complete and accurate
  • Technical approach validated
  • Requirements testable
  • Throughline resolved
Checkpoint features: Configurable checklists, notes field, approval tracking (who + when), staleness detection (flags if new documents added since approval), and reset capability.
Convergence: Three Output Document Types
PRD
For: Build / development initiatives
  • Executive Summary
  • Problem Statement & Goals
  • Requirements (functional + non-functional)
  • User Stories
  • Technical Considerations
  • Risks & Mitigations
Evaluation Framework
For: Vendor / tool / platform selection
  • Weighted criteria matrix
  • Platform comparison table
  • Scoring methodology
  • Recommendation with rationale
  • Risk/benefit assessment
Decision Framework
For: Governance / policy / strategic decisions
  • Decision context
  • Stakeholder analysis
  • Options analysis with trade-offs
  • Risk/benefit assessment
  • Recommended path forward
All output types include: Value alignment confirmation (ties to KPIs), tool/platform recommendations (simplest effective tool), AI risk & compliance review (EU AI Act, data classification), and an evaluation/QA plan.
Multi-Pass Synthesis (Optional, Higher Quality)

Available for Intelligence and Synthesis stages. Runs 3 parallel analyses at different temperatures, then combines the best insights with a meta-synthesis pass.

Meta-Synthesis (Opus)

Trust hierarchy for meta-synthesis:

  • Conservative (A): Most trusted for facts, data points, direct evidence
  • Balanced (B): Trusted for patterns and connections
  • Exploratory (C): Most trusted for novel insights and creative connections
Trade-off: ~4x runtime for meaningfully higher quality. Recommended for important initiatives where insight depth matters.
Operationalize: Three Paths to Action
1. Direct Project Creation

Create a project directly from any approved proposed initiative. No PRD required.

  • Quick path for clear initiatives
  • IFU scores map to project scores
  • Status: identified
  • Available from Proposed Initiatives tab
2. AI-Extracted Project

AI analyzes approved document and extracts project details automatically.

  • Title, description, department
  • ROI, effort, alignment scores
  • Initial task list
  • Confidence indicators
3. Tasks from State Changes

Create tasks directly from the "So What?" section of the convergence output.

  • Select which state changes become tasks
  • Optionally link to a project
  • "Next Human Action" as high-priority task
  • Full traceability to source
