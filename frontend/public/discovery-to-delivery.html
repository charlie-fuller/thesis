Discovery to Delivery

The end-to-end journey from a raw question to a delivered outcome

1. Discover
Spot an opportunity or problem
Something catches your attention -- a meeting insight, a stakeholder request, a market signal, a recurring complaint. This is the raw material.
You
Notice a signal Meeting note, article, Slack thread, or gut feeling about an opportunity
Gather evidence Upload docs, sync vault notes, collect relevant materials into KB
Frame the question What problem are we solving? What do we believe? What don't we know?
System
Process documents Auto-classify, embed, tag, and organize uploaded materials
Vault sync Pull new local vault notes automatically into KB
Surface connections Show related docs, similar initiatives, relevant stakeholders
Agents
Atlas (Strategy) Help frame the opportunity in strategic context
Sentinel (Research) Provide market and industry context
Discovery Guide Suggest investigation framing (ps, h, g items)
You have enough context to define a formal initiative
2. Scope
Create an initiative and link evidence
Formalize the opportunity into a DISCO initiative. Link the document folders, define the investigation framing, and identify stakeholders.
You
Create initiative Name, description, target department, value alignment
Link document folders Select KB folders containing relevant evidence
Assign stakeholders Identify sponsor, champions, and key participants
System
Create initiative record Status: draft, ready for pipeline
Index linked docs Make all linked documents available to pipeline agents
Track relationships Link stakeholders, documents, and initiative in graph
Agents
Discovery Guide Extract suggested framing from linked documents (auto-triage)
Navigator Suggest relevant stakeholders based on document content
-- Waiting for pipeline trigger
Initiative created, documents linked, ready for pipeline
3. DISCO Pipeline
Run the 4-stage agent pipeline
Each stage has a dedicated AI agent and a human checkpoint. Agents analyze, you review and approve. The pipeline produces actionable output documents.
You
Review at each checkpoint Approve, reject, or request changes at gates 1-4
Conduct discovery sessions Interviews, workshops, stakeholder conversations (human-led)
Select output type PRD, Evaluation Framework, or Decision Framework
System
Checkpoint gates Enforce mandatory review between each stage
Status progression draft > triaged > in_discovery > consolidated > synthesized > documented
Traceability Every output traces back to source documents through the spine
Agents
Discovery Guide Triage (GO/NO-GO), session planning, coverage assessment
Insight Analyst Evidence extraction, hypothesis mapping, decision document
Initiative Builder + Req Gen Bundle initiatives, score IFU, generate PRD/eval/decision docs
Approved output document ready for operationalization
4. Operationalize
Turn documents into projects and tasks
Create projects from DISCO outputs, extract tasks from state changes, score and prioritize in the project pipeline.
You
Create project Direct creation or AI-extracted from approved document
Review extracted tasks Accept, modify, or reject suggested tasks from state changes
Score and prioritize Set ROI, effort, alignment scores; assign tier (T1-T4)
System
Link traceability Project traces back to initiative, documents, and insights
Calculate scores Composite scoring across dimensions, tier assignment
Create task records Tasks linked to project and source DISCO output
Agents
AI extraction Parse document for title, description, tasks, scores
Kraken Evaluate tasks and suggest agent-assisted automation
-- Available on demand via chat
Project in pipeline, tasks on Kanban board
5. Execute
Work through tasks on the Kanban board
Assign owners, set due dates, track progress. Use agents for assistance. Move tasks through the board.
You
Assign and schedule Set owners, due dates, priorities for each task
Execute work Complete tasks, update status, resolve blockers
Track dependencies Manage task dependencies and sequencing
System
Kanban tracking Backlog > To Do > In Progress > Done
Progress rollup Task completion rates roll up to project status
Notifications Due date reminders, blocked task alerts
Agents
Kraken AI recommendations for task approach and automation
Any agent via chat Research, draft content, analyze options on demand
-- Agents available as needed
Tasks completed, outcomes measurable
6. Deliver
Measure outcomes and feed learnings back
Close the loop. Capture what worked, what didn't, and what you learned. Feed insights back into KB for future initiatives.
You
Assess outcomes Did the project deliver on its value alignment and KPIs?
Capture learnings Write retrospective notes, update vault documentation
Identify next opportunities What new questions emerged? What should be investigated next?
System
Archive initiative Move to archived status with full audit trail
Sync learnings to KB New documents flow back via reverse vault sync
Update graph Relationship data enriched with outcomes
Agents
Atlas Help contextualize outcomes in broader strategy
-- Ready for next cycle
-- Ready for next cycle
The Feedback Loop

Delivery is not the end -- it is the beginning of the next discovery. Learnings captured in Phase 6 become source documents for future initiatives, creating a continuous improvement cycle.

Deliver › Learnings › KB › New Signal › New Initiative › Pipeline › Execute › Deliver
Compounding value: Each cycle enriches the Knowledge Base, making future discoveries faster and more evidence-based. Agent responses improve as KB context grows. Stakeholder intelligence accumulates. The platform gets smarter with use.
