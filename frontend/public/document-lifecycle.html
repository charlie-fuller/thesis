Document Lifecycle

How documents flow from ingestion to agent-informed insights

Ingestion Processing Organization Linking Usage
1

Ingestion

Entry Points

How documents enter the system -- four distinct pathways feeding the knowledge base.

Obsidian Vault Sync

Automatic, bidirectional sync powered by a local launchd agent. Watches your Obsidian vault and uploads new or changed markdown files to the platform. Runs continuously in the background.

Manual Upload

Drag-and-drop interface in the Knowledge Base. Supports markdown, PDF, and plain text files. Preserves original filename and assigns to the selected folder path.

Agent-Generated Documents

The Discovery Agent can save research documents directly to the KB using an inline save command. Documents are tagged with their source agent and linked initiative.

Reverse Sync

Generated documents flow back to the local Obsidian vault. The sync agent pulls docs flagged with needs_reverse_sync and writes them to the appropriate vault folder.

2

Processing

Transformation

Raw documents are transformed into searchable, embeddable, and classifiable units.

Text Extraction

Content is extracted from markdown, PDF, and plain text formats. Markdown frontmatter is parsed separately and preserved as structured metadata.

Content Chunking

Documents are split into overlapping chunks (800 characters with 200-character overlap) to optimize for embedding quality and retrieval precision.

Embedding Generation

Each chunk is embedded using Voyage AI to produce dense vector representations. These enable semantic search -- finding documents by meaning, not just keywords.

Auto-Classification

Documents are automatically tagged across 18 tag categories. Tags are inferred from content analysis and can also be manually assigned or corrected after the fact.

3

Organization

Structure

Documents are organized into a navigable, searchable structure that mirrors your existing workflows.

Folder Hierarchy

Folder paths from the Obsidian vault are preserved. The KB presents a tree view built client-side from flat folder paths, with recursive document counts at each level.

Tag-Based Categorization

Both auto-assigned and manual tags provide a cross-cutting dimension for filtering. Tags enable discovery across folders -- finding related content regardless of where it lives.

Source Tracking

Every document records its origin: Obsidian sync, manual upload, or agent-generated. This provenance is visible in the KB interface and used for filtering and audit.

Full-Text and Semantic Search

Keyword search (via Supabase ilike) and semantic search (via Voyage AI embeddings) work together. Results can be filtered by tags, source, folder, and sort order.

4

Linking

Connections

Documents connect to initiatives, projects, and stakeholders -- forming the graph that agents reason over.

DISCO Initiative Folders

Link entire folders of documents to a DISCO initiative via disco_initiative_documents. Agents in the DISCO pipeline analyze all linked docs at each stage.

Project Documents

Individual documents can be linked to projects via project_documents. This provides per-project evidence and context that agents reference during analysis.

Stakeholder References

Content is scanned for references to known stakeholders. These implicit links surface in the stakeholder graph, connecting people to the evidence that mentions them.

Cross-References

Documents that reference or overlap with existing KB content are identified. This enables agents to pull in related context even when it was not explicitly linked.

5

Usage

Agent Consumption

Agents across the platform draw on documents to produce grounded, citation-backed analysis.

KB-Grounded Responses

All 22 agents prioritize KB content over general knowledge. Responses include citations linking back to source documents, ensuring traceability and trust.

DISCO Pipeline Analysis

Each DISCO stage (Discovery, Intelligence, Synthesis, Convergence) analyzes the full set of linked documents. Evidence tables with direct quotes are built during Intelligence.

Meeting Room Context

Meeting rooms can reference KB documents as shared context. Agents in autonomous discussion mode draw on this material to inform their contributions.

Throughline Mapping

Insights and recommendations map back to their source documents via throughlines. This preserves the chain of evidence from raw input to final strategic output.

Bidirectional Loop: Reverse Sync

Documents generated by agents (Stage 1, pathway 3) are flagged with needs_reverse_sync = true. The local sync agent periodically pulls these documents and writes them to the corresponding Obsidian vault folder. This completes the bidirectional loop: vault content flows into the platform for analysis, and platform-generated insights flow back into the vault as first-class documents. State is tracked per-document via reverse_synced_at timestamps.
