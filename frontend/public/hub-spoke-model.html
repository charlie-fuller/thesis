Governance Checklist
Before any tool/agent goes live, confirm each item -- this is the hub's quality gate
Documentation

MD file with purpose, usage, and architecture. JSON config if applicable. KB entry created.

Check: Can someone unfamiliar maintain this from docs alone?

Ownership

Primary owner and backup owner assigned. Both have access and knowledge. Handoff doc exists.

Check: If owner goes on PTO tomorrow, who takes over?

Security

Enterprise agreement active. Auth beyond password. No external data sharing. Data classification applied.

Check: Would this pass a security audit today?

Evaluation

Success criteria defined. Testing cadence set. QA process documented. Performance baseline recorded.

Check: How do we know if this stops working correctly?

Compliance

German Works Council cleared (if applicable). Data residency confirmed. Regulatory requirements met.

Check: Are there regional or legal constraints we haven't addressed?

Lifecycle

Sunset criteria defined. Review date scheduled. Upgrade/migration path considered.

Check: When should we re-evaluate if this is still the right tool?
