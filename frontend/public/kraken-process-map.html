Five Evaluation Dimensions (0-20 Points Each)

Each task is scored on five dimensions. The total (0-100) determines the confidence level and category. Kraken uses project context, linked KB documents, and web search to inform its assessment.

1
Information Sufficiency

Is there enough information in the task description, project context, KB docs, and web to complete the work?

0: Needs proprietary data20: All info available
2
Output Clarity

Is the expected deliverable clear? A defined artifact scores high; vague "improve X" scores low.

0: Undefined output20: Clear artifact
3
Execution Feasibility

Can the work be done with text generation, KB context, and web research? Or does it need system access?

0: Needs systems20: Pure text work
4
Completeness Achievable

Can AI produce a usable end result? Or just a starting skeleton that requires significant human work?

0: Outline only20: Full deliverable
5
Domain Fit

Is this the type of work AI excels at? Research, analysis, and drafting score high. Physical tasks score zero.

0: Physical/political20: Research/analysis
Three-Tier Categorization
Automatable

AI Can Complete This

70-100% Confidence

AI can complete this task fully and independently with high confidence.

Strong Indicators
  • Research synthesis and information gathering
  • Market research, competitor analysis
  • Documentation drafting
  • Comparative analysis and evaluation tables
  • Recommendation generation
  • Template and framework creation
  • Vendor/tool landscape reviews
  • Meeting preparation briefs
Assistable

AI Can Help Significantly

40-69% Confidence

AI can make meaningful progress but needs human review and input to finalize.

Moderate Indicators
  • Strategy formulation (AI drafts, human validates)
  • Technical architecture proposals
  • Budget estimation frameworks
  • Stakeholder mapping from documents
  • Content requiring brand voice
  • Analysis requiring organizational context
Manual

Requires Human Judgment

<40% Confidence

Requires human judgment, system access, or real-world action that AI cannot provide.

Strong Indicators
  • Physical actions or real-world interactions
  • Decisions requiring organizational politics
  • Internal system access (Jira, CRM, etc.)
  • Negotiations, approvals, sign-offs
  • Tasks dependent on proprietary live data
  • Purchasing or financial transactions

Agenticity Score: Project-Level AI Readiness

The agenticity score is a project-level metric showing what percentage of tasks AI can handle. Assistable tasks count at 50% weight since they still need significant human involvement.

(automatable + 0.5 x assistable) / total x 100

Displayed as a percentage in the Kraken panel and project card

Example: 10-Task Project
4 Automatable 3 Assistable 3 Manual

(4 + 0.5 x 3) / 10 x 100 = 55.0%

More than half of this project's work can be AI-assisted or fully automated.

What Happens During Execution
1. Context Assembly

For each selected task, Kraken assembles:

  • Full project description and state
  • Task title, description, notes
  • Linked KB documents (up to 10)
  • Initiative documents (up to 15)
2. Work Production

AI generates substantive work product:

  • Uses web search for real-time data
  • Structured with headers and bullets
  • Cites KB sources and web findings
  • Flags [USER INPUT NEEDED] items
3. Output Delivery

Work is delivered non-destructively:

  • Always: Added as task comment
  • If >200 words: Also saved to KB
  • KB docs tagged: kraken, auto-generated
  • KB docs linked to project
4. Next Steps

Every execution output ends with:

  • Summary of what was produced
  • Items requiring human decision
  • Suggested next actions
  • Quality confidence assessment
Non-destructive guarantee: Kraken never changes task status, deletes tasks, or modifies existing content. All output is additive - task comments and new KB documents. You review and decide what to use.
Safety Guardrails
What Kraken Cannot Do
  • Access external systems (Jira, Salesforce, CRM)
  • Send emails or Slack messages
  • Schedule meetings or make purchases
  • Execute code or technical operations
  • Access internal spreadsheets or dashboards
  • Make organizational decisions
Transparency Guarantees
  • Every evaluation includes reasoning
  • Every execution ends with next steps
  • [USER INPUT NEEDED] flags for human decisions
  • Confidence score visible per task
  • Real-time streaming shows progress
  • Execution results reviewable before use
Staleness Detection
  • Task hash computed from all task IDs + update times
  • Amber "Stale" badge if tasks changed since evaluation
  • Re-evaluation required before new execution
  • Prevents acting on outdated assessments
Rate Limits
  • Max 5 web searches per evaluation
  • Max 5 web searches per task execution
  • 4096 token output limit per task
  • Manual task selection required (no auto-execute)
When to Use Kraken
Best Suited For
  • Projects with many research/documentation tasks
  • Early-stage projects where groundwork needs doing
  • Portfolio prioritization (compare agenticity scores)
  • Tasks created from DISCO state changes
  • Pre-meeting preparation research
  • Competitive landscape analysis
Less Effective For
  • Projects with mostly execution/implementation tasks
  • Tasks requiring internal system access
  • Tasks dependent on live conversations or approvals
  • Work requiring creative direction or brand judgment
  • Tasks with very little description or context
