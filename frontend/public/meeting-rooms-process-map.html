Two Discussion Modes

Guided Discussion Default

You ask questions, the Facilitator routes them to the right agents. Interactive, back-and-forth conversation.
  • You drive the conversation with questions
  • Facilitator selects 1-3 relevant agents per turn
  • Agents respond sequentially (50-100 words each)
  • Each agent sees what previous agents said
  • Use @agent to speak directly to a specific agent
  • Ask for a summary anytime

Autonomous Discussion On Demand

You provide a topic and agents discuss it among themselves across multiple rounds. You observe and can interject.
  • You set the topic and number of rounds (1-5)
  • All agents speak once per round (~75 words)
  • Agents build on each other's points
  • Speaking order: priority-based or round-robin
  • You can interject anytime to pause and redirect
  • Final round focuses on synthesis and takeaways

Step 2b: Autonomous Discussion Flow

Autonomous mode lets agents discuss a topic among themselves across multiple rounds. You observe the conversation unfold in real-time and can interject at any point.

Discussion Begins
Round 1: Establish Positions
"Initial positions - state your perspective clearly"
  • Each agent speaks once in priority order
  • First speaker sets the frame
  • Later speakers see earlier responses
  • ~75 words per agent
Round 2+: Develop and Challenge
"Build on, question, or challenge what you've heard"
  • Agents reference each other by name
  • Challenge assumptions, connect ideas
  • New evidence and perspectives emerge
  • Each agent sees all prior round messages
Final Round: Synthesize
"Synthesize and conclude - actionable recommendations"
  • Agents provide their key takeaways
  • Actionable recommendations surface
  • Areas of agreement and disagreement clarified
  • Discussion auto-completes after this round
User Interjection: Type a message at any time during autonomous discussion. The discussion pauses immediately, your message is processed, and you return to guided mode. You can start a new autonomous discussion anytime after that.
Speaking Order Options
Priority Order (Default)

Agents speak in the order they were added to the meeting. Same order every round. Useful when you want a consistent flow.

Round Robin

Starting position rotates each round. No agent always speaks first. Useful for balanced influence across perspectives.

The Three Roles in Every Meeting
Meta-Agent

Facilitator

Manages conversation flow. Never provides domain expertise - only orchestrates.

Responsibilities
  • Greets and sets the stage
  • Routes questions to relevant agents
  • Ensures balanced participation
  • Provides conversational bridges between agents
  • Detects greetings and holds before inviting agents
Routing Logic
  • Keyword matching against agent specialties
  • Max 3 agents per turn
  • Force-includes essential perspectives (Sage, Nexus)
  • Skipped entirely for @mentions
Meta-Agent

Reporter

The sole voice for summaries. Prevents multiple agents from each giving their own recap.

Triggers
  • "Give me a summary"
  • "What are the key takeaways?"
  • "Summarize our discussion"
  • "Action items", "next steps", "wrap up"
Output Style
  • Uses domain labels, not agent names
  • "Financial analysis shows..." not "Capital said..."
  • Preserves disagreement (no false consensus)
  • Shorter than the source discussion
Specialists

Your Chosen Agents

Domain experts selected when you create the meeting. Each brings a distinct perspective.

How They Work
  • Speak only when Facilitator invites them
  • See full message history + KB context
  • See what other agents said this round
  • 50-100 words (guided) or ~75 words (autonomous)
Essential Perspectives
  • Sage (people/change) - auto-included when present
  • Nexus (systems thinking) - auto-included when present
Context Injection: What Agents Know

Every time you ask a question (or agents discuss in autonomous mode), the system automatically retrieves relevant context from two sources and injects it into each agent's prompt.

Source 1

Knowledge Base (Vector Search)

Voyage AI embeddings search your entire KB for the most relevant content.

What's Retrieved
  • 10 most relevant document chunks
  • Similarity scored (higher = more relevant)
  • Includes Obsidian vault files, uploads, transcripts
  • Includes past conversation excerpts
Agent Instruction

"If KB context addresses the question, you MUST reference it specifically. Quote or paraphrase relevant passages and cite the source."

Source 2

Knowledge Graph (Neo4j)

Structured relationship data about your organization's stakeholders and concerns.

What's Retrieved
  • Top 3 stakeholders with roles + sentiment
  • Top 3 concerns with severity levels
  • Top 2 ROI opportunities with status
  • Key relationships between entities
How Agents Use It

Agents reference specific people, concerns, and opportunities by name. "Given that [Stakeholder] has expressed concern about..."

Transparency: After each agent response, click "Sources" to see exactly which KB documents and graph entities informed the answer. Each KB source shows a similarity percentage.
Special Interactions
@Mention Direct Routing

Type @capital to speak directly to Capital. Bypasses Facilitator entirely. Agent responds alone with full context. Supports multiple mentions: @capital @guardian.

Summary Request

Ask for a "summary", "recap", "takeaways", "action items", or "next steps". Reporter responds exclusively - other agents stay silent. Uses domain labels for external-ready output.

Autonomous Interjection

During autonomous discussion, type any message to immediately pause the discussion. Your message is processed, and the meeting returns to guided mode. Start autonomous again anytime.

Export to KB

Click "Export to KB" to save the entire meeting transcript as a Knowledge Base document. This makes the discussion searchable and available as context in future conversations.

Key Principles
★
Multi-Perspective

Every topic benefits from multiple viewpoints. Mix financial, technical, people, and governance lenses.

⚖
Facilitator as Gatekeeper

Facilitator manages flow, never provides domain answers. Ensures no agent dominates.

⚙
Context-Grounded

Agents cite your KB documents and stakeholder data. Answers are grounded in your organization's reality.

↺
Challenge Over Agreement

Agents are instructed to question assumptions and challenge each other, not just agree.

☶
Essential Perspectives

Sage (people) and Nexus (systems) auto-join when present to prevent blind spots.

✎
You Stay in Control

Even in autonomous mode, one message pauses everything. You drive, agents advise.
