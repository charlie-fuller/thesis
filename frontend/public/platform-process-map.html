Governance Evaluation Criteria
Five criteria for evaluating platform suitability -- from Raza (Business Platforms), validated with enterprise IS/Finance requirements
1. Reliability

Does the platform consistently deliver correct, predictable results? Can we trust the output for business decisions?

Ask: "Can we rely on this for financial reporting? For customer-facing output?"

2. Governance

Does the platform enforce policies, permissions, and access controls? Can we restrict what data flows where?

Ask: "Can PII be filtered? Can we enforce data boundaries between departments?"

3. Traceability

Can every request and response be tracked with correlation/trace IDs? Can we reconstruct what happened?

Ask: "If something goes wrong, can we trace the exact path and inputs?"

4. Auditability

Does the platform produce audit-ready logs? Can we satisfy SOC 2 Type 2, user access reviews, and release change management?

Ask: "Can we generate audit reports showing who changed what, when, and why?"

5. Fault Tolerance

What happens when the platform fails? Is there graceful degradation, retry logic, and clear error handling?

Ask: "If the AI service is down, does the workflow fail safely or corrupt data?"

Key Questions at Each Decision Stage
Ask these questions during discovery to narrow platform selection -- sourced from Feb 13 discussions with Raza and Steffen
Stage 1 Solution Classification
  1. Is this searching existing information or creating new outputs?
  2. Does it need to take actions (write data, trigger processes)?
  3. Is "close enough" acceptable, or must results be deterministic?
  4. Could this be solved with a process change instead of a build?
  5. What does the current process look like? (Map it before building.)
Source: Steffen -- "Sometimes the process is just bad. You can avoid a lot of work by just refining the process."
Stage 2 Data Assessment
  1. What data sources are needed? Are they already in Glean?
  2. If not in Glean, is there an API we can build a connector for?
  3. How large is the context? (128K = Glean, 200K = Claude, 2M = Gemini)
  4. Are there large files (video, PDFs, codebases)?
  5. Could we build a Glean connector to an MCP server for this data?
Source: Steffen -- "We can build custom connectors. We've done it before with Chorus in a pretty short amount of time."
Stage 3 Context & Complexity
  1. Will this require multi-step reasoning or just retrieval?
  2. How curated/narrow is the document set? (Glean works best with narrow scope.)
  3. Do we need to analyze video or multimedia? (Gemini/NotebookLM only.)
  4. Does the output need to be formatted specifically (slides, SOPs, structured data)?
  5. Is this something that needs to happen automatically on a schedule?
Source: Platform inventory -- "Glean broad queries fail. Only works for narrow, curated domains."
Stage 4 Audience & Deployment
  1. How many people need access? (Company-wide = Glean/Gemini default.)
  2. Are German employees included? (Limits tool options.)
  3. Is this shared within a team or individual productivity?
  4. Do external partners or customers need access?
  5. Does the team already use Google-native workflows?
Source: Steffen -- "Gems can be shared within the whole company. People are not aware this is a possibility."
Stage 5 Governance Requirements
  1. Does this touch SOC 2 audit scope? (User access reviews, change management?)
  2. Do we need full request/response logging with trace IDs?
  3. Must we demonstrate who changed what, when, and why?
  4. Does this involve PII that needs to be filtered in transit?
  5. Is there a regulatory requirement (GDPR, financial compliance)?
Source: Raza -- "Governance, traceability, auditability, fault tolerance -- these are key evaluation criteria."
Stage 6 Operations & Maintenance
  1. Who will maintain this if the builder leaves the company?
  2. Is there a backup owner identified and committed?
  3. What evaluation/testing cadence will be established?
  4. What's the response time expectation if it breaks?
  5. Are standard configuration files (MD, JSON, KB docs) provided?
Source: Steffen -- "Somebody was on PTO, the script was needed, nobody had access. It happened recently."
Stage 7 Compliance & Approval
  1. What is the data classification level? (Internal, customer, PII?)
  2. Will German employees use this? (Works Council approval needed for new tools.)
  3. Is there an enterprise agreement with the vendor?
  4. Does data leave our controlled environment?
  5. Has the experimentation request process been followed?
Source: Steffen -- "The biggest risk is our data. If they use new tools without enterprise agreements, data could train models."
