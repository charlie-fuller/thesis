The 4 Scoring Dimensions

Each dimension captures a different aspect of project viability. Higher is always better (effort is inverted: 5 = minimal effort).

📈 ROI Potential

5Transformative revenue or savings impact
4Significant measurable business value
3Moderate value, clear but limited impact
2Marginal value, hard to quantify
1Minimal or speculative value

⚙ Effort Level (inverted)

5Quick win: days, minimal resources
4Small project: 1-2 weeks, one team
3Medium project: weeks, some coordination
2Large project: months, cross-team
1Massive: quarters, org-wide change

🎯 Strategic Alignment

5Core to company strategy, executive mandate
4Strongly aligned with strategic priorities
3Moderately aligned, supports goals
2Loosely related to strategy
1Not aligned with current strategy

🚀 Org Readiness

5Champion in place, team ready, budget approved
4Sponsor identified, resources available
3Some support, resources partially available
2Interest but no commitment or resources
1No sponsor, resistance, or competing priorities

Tier Classification

The total score maps to a tier that determines project priority. Tiers are computed automatically in the database.

Tier 1
17 - 20 points
Quick wins and strategic priorities. High value, low effort, strong alignment. Act immediately.
Tier 2
14 - 16 points
Strong opportunities worth pursuing. Plan and resource within the quarter.
Tier 3
11 - 13 points
Worth tracking but not urgent. May need conditions to change before acting.
Tier 4
4 - 10 points
Low priority or not viable now. Park or revisit when circumstances change.
CASE WHEN triage_score >= 17 THEN 'T1' WHEN triage_score >= 14 THEN 'T2' WHEN triage_score >= 11 THEN 'T3' ELSE 'T4' END

Confidence Score

Independent of the triage score, each project has a confidence score (0-100) reflecting how certain you are about the scoring inputs. Higher confidence means the triage score is more reliable.

📚 Evidence Quality 25 pts

How strong is the data backing the scores? Direct measurements, customer research, or just assumptions?

👥 Stakeholder Validation 25 pts

Have key stakeholders reviewed and agreed with the assessment?

📈 Precedent Exists 25 pts

Has something similar been done before? Is there a track record to reference?

🔍 Scope Clarity 25 pts

How well-defined are requirements, timeline, and success criteria?

How Projects Are Created

Projects can enter the system through four paths. DISCO Convergence is the recommended path for well-researched opportunities.

🔎 DISCO Convergence

Best path: initiative analysis produces a validated project proposal with pre-filled scores based on discovery evidence.

Discovery → Intelligence → Synthesis → Convergence → Project

📝 Manual Creation

Create directly from the Projects page. You fill in all details and scores yourself.

Projects page → New Project → Score → Save

💬 Chat Suggestion

Agents like Oracle or Strategist can recommend creating a project from analysis. You confirm and score.

Chat → Agent analysis → Create project → Score

🎤 Meeting Action Item

Meeting notes may surface opportunities that become projects via the Granola pipeline.

Meeting → Granola sync → Action item → Project

Key Principles

📈 Score Honestly

Inflated scores undermine the system. Use the confidence rubric to flag uncertainty rather than guessing high.

🏆 Tiers Guide Action

T1 projects get immediate attention. T4 projects aren't bad -- they're just not ready yet. Conditions change.

🔄 Status Reflects Reality

Update status as projects progress. The Pipeline view depends on accurate status for workload management.

🔗 Link Everything

Attach relevant KB documents, connect tasks, and reference stakeholders. Context makes scoring more reliable.

🔁 Re-score When Conditions Change

Scores aren't permanent. When new information arrives or circumstances shift, update the scores and tier.

📚 Evidence Beats Intuition

DISCO-originated projects start with better scores because they're backed by document analysis and structured research.
