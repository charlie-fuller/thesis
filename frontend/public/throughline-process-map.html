What is a Throughline?

A throughline is your investigation framing - the structured set of problem statements, hypotheses, and gaps that give every DISCO agent a consistent lens for analysis. Think of it as the "research questions" for your discovery.

Without a throughline, agents analyze documents in isolation. With one, every agent maps its findings back to your specific questions, and the final Convergence stage resolves each item with evidence.

You don't need to define it upfront - the Discovery Guide can extract one from your documents automatically.

Before Throughline

"Analyze these documents about AI governance."

Agent produces generic analysis

With Throughline

"Analyze these documents to test whether [h-1] German Works Council approval will block GenAI deployment, address [g-1] missing data classification policy, and resolve [ps-1] unclear governance ownership."

Agent maps every finding to your specific questions

Anatomy: Four Components
Problem Statements

What's Wrong?

ps-1, ps-2, ps-3 ...

Defines the specific problems the initiative is investigating. Each problem statement anchors the analysis.

  • Clear, specific problem definition
  • Scoped to what you can influence
  • Referenced by ID in all agent outputs
Hypotheses

What Do We Believe?

h-1, h-2, h-3 ...

Testable assumptions, beliefs, or predictions that agents seek to confirm or refute with evidence.

  • Assumption: Taken as true, needs validation
  • Belief: Team conviction, may be wrong
  • Prediction: Expected outcome if X happens
Gaps

What Don't We Know?

g-1, g-2, g-3 ...

Known unknowns that discovery sessions and document analysis should address.

  • Data: Missing metrics or evidence
  • People: Stakeholders not yet consulted
  • Process: Undefined workflows
  • Capability: Unknown technical feasibility
Desired Outcome

What Does Success Look Like?

Free text

A clear description of the desired end state. Agents use this to evaluate whether proposed solutions actually solve the problem.

  • Measurable where possible
  • Anchors "So What?" analysis
  • Guides convergence recommendations
Three Ways to Define a Throughline
1. Agent-Extracted (Recommended)
Best for: Documents already linked

Run the Discovery Guide on your linked documents. It analyzes content and extracts framing items automatically.

  1. Link documents or folders to your discovery
  2. Run the Discovery Guide (auto-detects Triage mode)
  3. Agent outputs a "Suggested Framing" section
  4. A review panel appears with extracted items
  5. Accept all, select individually, or dismiss
  6. Accepted items populate the throughline
2. Chat-Based
Best for: Exploring ideas interactively

Chat with the Discovery Agent and ask for help framing. It proposes items as interactive cards you can select.

  1. Open Chat from your discovery
  2. Ask: "Help me frame this discovery"
  3. Agent proposes problem statements, hypotheses, gaps
  4. Items appear as selectable cards
  5. Check the items you want to keep
  6. Click "Apply to Discovery" to save
3. Manual Entry
Best for: When you know exactly what to investigate

Directly add items through the create/edit modal. Full control over every component.

  1. Create or edit a discovery
  2. Expand "Investigation Framing" section
  3. Click "+ Add" for problem statements, hypotheses, or gaps
  4. Enter the text and select type (for hypotheses/gaps)
  5. IDs are auto-assigned (ps-1, h-1, g-1)
  6. Save to commit the throughline
You can combine methods: Start with agent-extracted framing from documents, then refine through chat, and add manual items for specific questions you want answered. The throughline can be edited at any point during the DISCO process.

Discovery

The Discovery Guide reads the throughline and uses it to focus analysis.

What It Does
  • Triage: Extracts suggested framing from documents (ps, h, g items)
  • Planning: Designs session questions targeting specific gaps (g-1, g-2)
  • Coverage: Reports which gaps are addressed and which remain open
Output References

"Evidence for h-1 found in [document]. g-2 remains unaddressed."

Intelligence

The Insight Analyst maps evidence directly to throughline items.

What It Does
  • Creates hypothesis-evidence mapping table
  • Identifies which hypotheses have supporting/contradicting evidence
  • Flags gaps with no new data
Output References

"h-2 (prediction): 3 sources support, 1 contradicts. Strength: moderate."

Synthesis

The Initiative Builder links proposed bundles to throughline items.

What It Does
  • Each proposed initiative references which ps/h/g items it addresses
  • Dependencies mapped to gap resolution order
  • Ensures no problem statement is left without an initiative
Output References

"Bundle A addresses ps-1 and h-1. Depends on g-3 resolution."

Convergence (Resolution)

The Requirements Generator produces the final resolution of every throughline item.

What It Does
  • Hypothesis resolution table (confirmed/refuted/inconclusive)
  • Gap status table (addressed/unaddressed/partially addressed)
  • State changes with owner and deadline
  • "So What?" analysis per item
Output References

"h-1: CONFIRMED. 4 sources confirm Works Council will engage. Next action: schedule briefing by March 15."

The "So What?" Analysis

For every resolved throughline item, Convergence answers three questions that turn analysis into action.

State Change Proposed

What should change as a result of this finding? Expressed as a concrete organizational shift, not an aspiration.

  • "Move from ad-hoc to policy-governed AI procurement"
  • "Shift data classification from manual to automated"
  • "Transition Works Council engagement from reactive to proactive"
Next Human Action

The single most important thing a human must do next. Becomes a high-priority task if you click "Create Tasks from State Changes."

  • "Schedule Works Council briefing by March 15"
  • "Request Zendesk export from Support team lead"
  • "Present data classification draft to CISO for review"
Kill Test

The condition under which you should stop pursuing this initiative. Prevents sunk-cost escalation.

  • "If Works Council rejects consultation framework by April"
  • "If ROI projection falls below 1.5x after real data analysis"
  • "If no executive sponsor confirmed within 30 days"
From analysis to tasks: Click "Create Tasks from State Changes" in the output viewer. Each state change becomes a task linked to the source discovery, with the Next Human Action flagged as high priority. Full traceability from task back to throughline item.
When to Define a Throughline
Define Throughline When:
  • You have specific questions to answer
  • Multiple stakeholders with different assumptions
  • The investigation could go in many directions
  • You need to prove/disprove specific beliefs
  • There are known unknowns you need to address
  • You want structured resolution at the end
Start Without Throughline When:
  • You're exploring a new area with no specific questions yet
  • You just want a general analysis of documents
  • The scope is narrow and well-defined already
  • You plan to let the Discovery Guide suggest framing first
No throughline? No problem. Run the Discovery Guide on your documents. If it detects content worth framing, it will suggest items. You can always add a throughline later.

Iterative refinement: The throughline is not locked after initial definition. You can add, edit, or remove items at any point. If you add new items mid-process, subsequent agents will pick them up. Re-run earlier stages if you want them to incorporate the updated framing.
Key Principles
★
Structured Lens

The throughline gives agents a consistent framework. Every finding maps back to your questions.

⚖
ID-Referenced

Every item has a unique ID (ps-1, h-1, g-1). Agents cite these IDs in their outputs for traceability.

↺
Progressive Resolution

Items accumulate evidence across stages. Resolution only happens at Convergence with full context.

⚙
Human Override

You can annotate any agent verdict. Your assessment is preserved alongside the agent's.

✓
Action-Oriented

"So What?" turns every finding into a state change, next action, and kill test.

☶
Optional but Powerful

DISCO works without a throughline. But with one, the output quality jumps significantly.
